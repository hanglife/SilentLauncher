## 创建计划任务：中文版 + 英文提示版

## 打开 PowerShell，复制执行

我保存了代码到 计划任务.txt 内

```

# ============================================================
# 创建“登录后启动”的计划任务
# Create a scheduled task that runs after user logon
# ============================================================

# 启动器程序路径 / Launcher executable path
$exe = "D:\Tools\SilentLauncher\SilentLauncher.exe"

# 配置文件路径 / Config JSON path
$config = "D:\Tools\SilentLauncher\launcher.json"

# 检查启动器是否存在 / Check whether launcher exists
if (-not (Test-Path $exe)) {
    throw "找不到启动器程序 / Launcher executable not found: $exe"
}

# 检查配置文件是否存在 / Check whether config file exists
if (-not (Test-Path $config)) {
    throw "找不到配置文件 / Config file not found: $config"
}

# 设置要执行的程序和参数
# Set executable and argument
$action = New-ScheduledTaskAction -Execute $exe -Argument "`"$config`""

# 设置触发条件：用户登录时运行
# Set trigger: run at user logon
$trigger = New-ScheduledTaskTrigger -AtLogOn

# 设置运行身份：当前登录用户，交互式运行，最低权限
# Set principal: current logged-in user, interactive mode, least privilege
$principal = New-ScheduledTaskPrincipal `
    -UserId "$env:USERDOMAIN\$env:USERNAME" `
    -LogonType Interactive `
    -RunLevel LeastPrivilege

# 设置任务参数：不限制运行时间，失败后重试 3 次，每次间隔 1 分钟
# Set task settings: no execution time limit, restart 3 times, 1 minute interval
$settings = New-ScheduledTaskSettingsSet `
    -ExecutionTimeLimit (New-TimeSpan -Hours 0) `
    -RestartCount 3 `
    -RestartInterval (New-TimeSpan -Minutes 1)

# 注册计划任务
# Register scheduled task
Register-ScheduledTask `
    -TaskName "SilentLauncher" `
    -Action $action `
    -Trigger $trigger `
    -Principal $principal `
    -Settings $settings `
    -Description "用户登录后隐藏启动指定程序 / Start selected apps hidden after user logon" `
    -Force

```

手动测试任务
创建完成后，直接运行一次测试：


```

# 手动启动任务 / Start task manually
Start-ScheduledTask -TaskName "SilentLauncher"

```
然后看 Kavita / Sideloadly / 你配置的软件有没有按预期启动。

## 查看相关计划任务

```

# 列出相关计划任务
# List related scheduled tasks
Get-ScheduledTask |
  Where-Object TaskName -match "SilentLauncher|Kavita|Sideloadly" |
  Select-Object TaskName,TaskPath,State

```


## 查看任务运行结果
```

# 查看任务运行结果
# Check scheduled task run result
Get-ScheduledTaskInfo -TaskName "SilentLauncher" |
  Select-Object TaskName,LastRunTime,LastTaskResult,NextRunTime,NumberOfMissedRuns

```


## 一般来说：


```LastTaskResult = 0```
代表运行成功。

如果不是 0，就说明启动失败、路径错误、权限不够，或者配置文件有问题。

## 删除任务

```
# 删除计划任务
# Remove scheduled task
Unregister-ScheduledTask -TaskName "SilentLauncher" -Confirm:$false
```

## 关键提醒
你不要改成 shell:startup。

计划任务比启动文件夹更稳，尤其是这种需要带参数、隐藏窗口、登录后延迟启动、失败重试的软件。


如果你的程序需要管理员权限，把这一行：

```
-RunLevel LeastPrivilege
```
改成：
```
-RunLevel Highest
```

并且用 <b>管理员 PowerShell</b> 执行创建任务。

但普通 Kavita / Sideloadly 自动启动，一般先用 LeastPrivilege 更安全。
